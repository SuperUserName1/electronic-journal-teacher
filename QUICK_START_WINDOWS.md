# Быстрый старт: Создание Windows EXE

## Самый простой способ - через GitHub Actions

### Шаг 1: Откройте GitHub репозиторий
Перейдите на https://github.com/ваш-username/ваш-репозиторий

### Шаг 2: Запустите сборку
1. Перейдите в раздел **"Actions"**
2. Выберите workflow **"Build Windows Release"**
3. Нажмите **"Run workflow"** (справа)
4. Выберите ветку (обычно `main`)
5. Нажмите зеленую кнопку **"Run workflow"**

### Шаг 3: Дождитесь завершения
- Сборка займет несколько минут
- Вы увидите прогресс в реальном времени

### Шаг 4: Скачайте результат
1. После завершения нажмите на завершенный workflow
2. Прокрутите вниз до секции **"Artifacts"**
3. Скачайте файл **"windows-build"**
4. Распакуйте архив - внутри будут файлы:
   - `ЭЖП: Электронный Журнал Преподавателя-Setup-1.0.0.exe` (установщик)
   - `ЭЖП: Электронный Журнал Преподавателя-Portable-1.0.0.exe` (портативная версия)

## Альтернатива: Автоматическая сборка при создании тега

```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub автоматически создаст релиз с Windows файлами.

---

**Примечание:** Локальная сборка Windows EXE на Linux невозможна из-за необходимости компиляции нативных модулей для Windows. Используйте GitHub Actions - это самый надежный способ.
